## Data Files

### `docker-images-list.json`
The main list of Docker official images with their metadata. This file is used to generate the README table above.

### `review/results.json`
A curated collection of **63 community Docker images** with their icons and metadata. This includes popular third-party images like:
- Development tools (GitLab, Drone, Harbor, Portainer)
- Databases (MinIO, CockroachDB, TimescaleDB)
- Media servers (Plex, Jellyfin, Navidrome)
- Home automation (Home Assistant, Node-RED)
- Self-hosted services (Nextcloud alternatives, VPN solutions)

All icon files for these images are located in `review/icons/` with proper file extensions matching the `logo_url` entries.

## Contributing

To add a new icon:
1. Add your icon image to the `icons/` directory
2. Update the `docker-images-list.json` file
3. Run `node update-readme.js` to regenerate this README
4. Use consistent naming: lowercase with hyphens (e.g., `my-app.png`)

For community images:
1. Add your icon to the `review/icons/` directory
2. Update the `review/results.json` file with the image metadata
3. Ensure the file extension in `logo_url` matches the actual file

## Icon Guidelines

- **Format**: PNG or SVG preferred (also supports WebP, JPG, GIF)
- **Size**: 512x512px recommended for PNG files
- **Naming**: Use lowercase with hyphens, format: `owner-image.ext` (e.g., `linuxserver-sonarr.svg`)
- **Quality**: High-quality, transparent backgrounds preferred
